# OOPs ke 4 pillars kya hote hain? Short explanation do.

Encapsulation → Data hiding + binding data & methods together using private fields and controlled access (getters/setters)

Abstraction → Exposing only essential features and hiding implementation details (using interface/abstract class)

Inheritance → Acquiring properties & behavior of parent class to promote code reuse

Polymorphism → Same interface, different behavior 
               → Compile-time (overloading) 
               → Runtime (overriding)


---

# Overloading vs Overriding

Overloading → same class, same method name, different parameters (compile-time)

Overriding → parent-child relation, same method signature, runtime decision (runtime polymorphism)

---

Abstraction → focus on WHAT to show (interface/abstract class), hides implementation

Encapsulation → focus on HOW to protect data (private fields + controlled access)

In one line

Abstraction hides implementation, Encapsulation hides data.

---

# Can we achieve abstraction without encapsulation? या encapsulation बिना abstraction possible है?


Short answer: **✔️ Correct — Encapsulation bina abstraction possible hai.**
But interviewer ko satisfy karne ke liye dono sides clear bolo 👇

---

### ✅ Final Interview Answer

```text id="ans1"
Encapsulation without abstraction → YES
We can hide data using private fields and expose getters/setters
without hiding implementation logic.

Abstraction without encapsulation → NO (practically)
Because to hide implementation, we need to hide data as well,
which requires encapsulation.
```

---

### 💻 Quick Code Proof

**Encapsulation (no abstraction):**

```java id="enc1"
class User {
    private int age;

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        if (age > 0) {
            this.age = age;
        }
    }
}
```

---

**Abstraction (uses encapsulation internally):**

```java id="abs1"
interface Payment {
    void pay(double amount); // abstraction
}

class UpiPayment implements Payment {
    private String upiId; // encapsulation

    public void pay(double amount) {
        System.out.println("Paid via UPI");
    }
}
```

# Interface vs Abstract class

1. Inheritance:
   Class → can extend only one abstract class
   Interface → can implement multiple interfaces

2. Methods:
   Abstract class → can have abstract + concrete methods
   Interface → methods are by default abstract (except default/static)

3. Variables:
   Abstract class → can have instance variables
   Interface → only public static final constants

4. Constructor:
   Abstract class → can have constructor
   Interface → cannot have constructor

5. Use case:
   Abstract class → when classes share common state/behavior
   Interface → when only contract is needed (no state)


---

# Interface me fields automatically public static final kyun hote hain?

Not because interface cannot be instantiated.

Actual reason:
Interface is meant to define a contract, not state.
So variables are constants → shared across all implementations.

Hence:
public → accessible everywhere
static → shared (no object needed)
final → constant (cannot change)

# Java me multiple inheritance kyun allowed nahi hai?

सही दिशा में हो — लेकिन यह answer अभी **too short** है. Interviewer यही पर probe करेगा: *“कैसी ambiguity?”*
चलो इसे strong बनाते हैं 👇

---

### ✅ Complete Answer (Diamond Problem)

```text id="dia1"
Java me multiple inheritance allow nahi hai due to ambiguity.

Suppose:
Class A has method show()
Class B and C extend A and override show()

Now class D extends B and C

D ko nahi pata kaunsa show() call kare (B ka ya C ka) → ambiguity
```

---

### ✅ Code Example (clear समझ)

```java id="dia2"
class A {
    void show() {
        System.out.println("A");
    }
}

class B extends A {
    void show() {
        System.out.println("B");
    }
}

class C extends A {
    void show() {
        System.out.println("C");
    }
}

// ❌ Not allowed in Java
class D extends B, C {
    public static void main(String[] args) {
        new D().show(); // ambiguity: B or C?
    }
}
```

---

### 🔥 Important Add-on (interview booster)

```text id="dia3"
Java supports multiple inheritance via interfaces
because interfaces don't have state (no instance variables),
and default method conflicts can be resolved explicitly.
```
