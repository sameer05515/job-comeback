#  difference between embedded and external server

### âœ… **Difference Between Embedded Server and External Server in Spring Boot**

| Feature                 | **Embedded Server**                       | **External Server**                         |
| ----------------------- | ----------------------------------------- | ------------------------------------------- |

| **Ease of Setup**       | Plug & play, **zero config**              | Requires server setup and deployment config |
| **DevOps Friendly**     | âœ… Yes (simple CI/CD pipelines)            | âŒ Extra steps needed                        |
| **Portability**         | High (run on any machine with Java)       | Medium (depends on external server setup)   |
| **Spring Boot Default** | âœ… Embedded Tomcat (or Jetty/Undertow)     | âŒ Not default, needs WAR packaging          |
| **Packaging**           | `spring-boot-maven-plugin` â†’ `jar`        | Traditional WAR with servlet container      |

# run app using embedded Jetty instead of Tomcat

To run a Spring Boot app using **embedded Jetty** instead of **default Tomcat**, follow these steps:

---

### âœ… 1. **Exclude Tomcat**

In your `pom.xml`, exclude **Tomcat** from `spring-boot-starter-web`:

```xml
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-web</artifactId>
  <exclusions>
    <exclusion>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-starter-tomcat</artifactId>
    </exclusion>
  </exclusions>
</dependency>
```

---

### âœ… 2. **Add Jetty Starter**

```xml
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-jetty</artifactId>
</dependency>
```

---

### âœ… 3. **Rebuild and Run**

```bash
mvn clean package
java -jar target/your-app.jar
```

---

### âœ… 4. **Verify**

Check logs on startup:


---

### ðŸ” Optional: Change Port in `application.properties`
